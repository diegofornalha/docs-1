O Cursor Chat permite que você faça perguntas ou resolva problemas na sua base de código com os modelos de linguagem mais capazes, tudo dentro do seu editor.

</Frame>

Para que os modelos de linguagem forneçam boas respostas, eles precisam saber coisas específicas que são relevantes para sua base de código — contexto.

O Cursor possui vários recursos integrados para fornecer contexto no chat, como incluir automaticamente o contexto em toda a sua base de código, pesquisar na web,
indexar documentação e referências especificadas pelo usuário para blocos de código. Eles são construídos para eliminar o tedioso copiar e colar necessário para trabalhar com modelos de linguagem em código.

Por padrão, o Cursor Chat está no painel de IA, que fica no lado oposto da sua barra lateral principal. Você pode alternar o painel de IA pressionando `Ctrl/⌘ + L`, que foca no chat quando aberto.
Para enviar sua consulta, pressione `Enter`.

### Mensagens do Usuário e da IA

As mensagens do usuário contêm o texto que você digita, junto com o contexto que você referenciou. Você pode voltar a qualquer mensagem anterior do usuário para editar e executar novamente suas consultas. Isso irá
sobrescrever quaisquer mensagens após isso e gerar novas.

As mensagens da IA são as respostas geradas a partir do modelo de IA que você escolheu. Elas são emparelhadas com a mensagem do usuário anterior. As mensagens da IA podem conter blocos de código analisados que podem ser adicionados à sua base de código com [aplicação instantânea](/chat/apply).

Todas as mensagens de usuário/IA juntas no mesmo thread são chamadas de thread de chat, e cada thread de chat é salva no seu histórico de chat.

### Histórico de Chat

Ao pressionar o botão "Chats Anteriores" no canto superior direito do painel de IA, ou pressionando `Ctrl/⌘ + Alt/Option + L`, você pode ver o histórico de chat. Você pode clicar em qualquer thread de chat para voltar e ver as mensagens que compõem esse thread,
e você também pode modificar o título do thread clicando no ícone de caneta, ou excluir o thread clicando no ícone de lixeira ao passar o mouse sobre o thread no histórico.

O título de um thread do Cursor é apenas as primeiras palavras da primeira mensagem do usuário.

### Contexto Padrão

Por padrão, o Cursor Chat inclui o arquivo atual como contexto. Você pode enviar uma consulta sem incluir nenhum contexto pressionando `Alt/Option Enter` ao enviar,
ou ativar `Padrão para nenhum contexto` em `Configurações do Cursor` > `Recursos` > `Chat`. Chats que começam com uma solicitação sem contexto não adicionarão contexto para nenhuma mensagem.

Conforme você digita, você pode ver o que será incluído no contexto nas pílulas abaixo da caixa de entrada.

### Adicionando Contexto

Por padrão, as mensagens do usuário conterão o texto que você digita, junto com o contexto que você referenciou. Você pode adicionar mais contexto personalizado a cada bolha com símbolos @, e por padrão,
o arquivo visualizado atualmente também será usado como contexto na mensagem do usuário.

Veja as páginas de [símbolos @](/context/@-symbols/@-files) para mais informações.

### Correção de IA no Chat

Um recurso conveniente para corrigir erros de lint na sua base de código é usar a correção de IA no chat. Para fazer isso, passe o mouse sobre o erro no editor e clique no botão azul de correção de IA que aparece.

O atalho de teclado para isso seria `Ctrl/⌘ + Shift + E`.

### Chat de Contexto Longo (Beta)

Você pode habilitar o chat de contexto longo indo para `Configurações do Cursor` > `Beta` > `Chat de Contexto Longo`. Com isso habilitado, você pode alternar entre diferentes modos de chat através de `Ctrl/⌘ .`.

O chat de contexto longo permite que você inclua pastas inteiras como contexto, já que os modelos suportados têm uma janela de contexto maior. Confira mais sobre os modelos apenas para contexto longo [aqui](/advanced/models#long-context-only-models).
