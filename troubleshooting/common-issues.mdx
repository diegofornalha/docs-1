title: Problemas Comuns
---

#### Vejo uma atualização no changelog, mas o Cursor não atualiza.

Se a atualização for muito recente, pode ser que ainda não tenha sido disponibilizada para você. Fazemos lançamentos em etapas, o que significa que liberamos novas atualizações para alguns usuários selecionados aleatoriamente antes de liberá-las para todos. Normalmente, alcançar todos os usuários leva cerca de 5 horas a partir do primeiro usuário. Para atualizações maiores, pode demorar mais.

Se você não quiser esperar, pode baixar o Cursor novamente em nossa [página inicial](https://cursor.com/). Provavelmente, você receberá a atualização mais recente instantaneamente. Não se preocupe, todas as suas configurações, extensões, etc., serão transferidas para a versão baixada novamente. Você não precisa desinstalar a versão existente primeiro.

#### Tenho problemas com meu login no GitHub no Cursor / Como faço para sair do GitHub no Cursor?

Você pode tentar usar o comando `Sign Out of GitHub` no painel de comandos `Ctrl/⌘ + Shift + P`.

#### Não consigo usar o GitHub Codespaces.

Infelizmente, ainda não oferecemos suporte ao GitHub Codespaces.

#### Tenho erros ao conectar ao SSH Remoto.

Atualmente, não oferecemos suporte para SSH em máquinas Windows. Se você não estiver usando uma máquina Windows, por favor, reporte seu problema para nós no [fórum](https://forum.cursor.com/). Seria útil incluir alguns logs para melhor assistência.

#### Cursor Tab e Cmd K não funcionam atrás do meu proxy corporativo.

Cursor Tab e Cmd K usam HTTP/2 por padrão, o que nos permite usar menos recursos com menor latência. Alguns proxies corporativos (por exemplo, Zscaler em certas configurações) bloqueiam HTTP/2. Para corrigir isso, você pode definir `"cursor.general.disableHttp2": true` nas configurações (`Cmd/Ctrl + ,` e depois procurar por `http2`).

#### Acabei de me inscrever no Pro, mas ainda estou no plano gratuito no aplicativo.

Tente sair e entrar novamente em [Configurações](/get-started/migrate-from-vscode#cursor-settings).

#### Quando meu uso será reiniciado novamente?

Se você está inscrito no Pro, pode clicar em `Gerenciar Assinatura` no [Painel](https://cursor.com/settings) e a data de renovação do seu plano será exibida no topo.

Se você é um usuário gratuito, pode verificar quando recebeu o primeiro e-mail nosso na sua caixa de entrada. Seu uso será reiniciado todo mês a partir dessa data.

#### Como desinstalo o Cursor?

Você pode seguir [este guia](https://code.visualstudio.com/docs/setup/uninstall) para desinstalar o Cursor. Substitua todas as ocorrências de "VS Code" ou "Code" por "Cursor", e ".vscode" por ".cursor".
